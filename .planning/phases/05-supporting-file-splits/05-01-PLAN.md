**Constructor signature:**
```js
constructor(game)
```
Store game reference (needed by replaceCharacterTokens for character name colors).

**Methods to extract from CombatUILog (move verbatim, then adapt):**
- replaceCombatTags(text) — simple semantic token replacement using COMBAT_TAGS
- replaceWrapperTags(text) — wrapper token replacement using WRAPPER_TAGS
- replaceCharacterTokens(text) — character name tokens with faction-colored spans
- replaceTooltipTags(text) — tooltip token replacement
- replaceArmorTokens(text) — armor token replacement with styled tooltips
- replaceWeaponTokens(text) — weapon token replacement with styled tooltips
- formatLogEntry(entry) — orchestrates all tag replacements, returns HTML string
- getAllCharacters() — helper returning [pc, ...npcs] (used by replaceCharacterTokens)

**Imports:** FACTIONS, GAME_CONSTANTS, ARMOR_TYPES, WEAPONS, COMBAT_TAGS, WRAPPER_TAGS from const.js

The formatter is nearly pure — all methods take text input and return HTML output. Only dependency is `game` for character color lookups in replaceCharacterTokens.

**In Game.js:**


**In CombatUILog.js:**
1. Remove all 8 extracted methods (replaceCombatTags, replaceWrapperTags, replaceCharacterTokens, replaceTooltipTags, replaceArmorTokens, replaceWeaponTokens, formatLogEntry, getAllCharacters)
2. Remove imports no longer needed (FACTIONS, ARMOR_TYPES, WEAPONS, COMBAT_TAGS, WRAPPER_TAGS — keep GAME_CONSTANTS for trimOldEntries)
3. Store formatter reference (from constructor or setDependencies)

   ```js

   ```
