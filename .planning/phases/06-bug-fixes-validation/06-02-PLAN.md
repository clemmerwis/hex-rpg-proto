**Area definition schema (from docs/reference.md):**
Required: id (string), name (string), background (string), width (positive number), height (positive number)
Optional: blocked (array of {q, r}), spawns (object with {q, r} values), npcs (array with templateId + hexQ + hexR), exits (array with id + hexes + target + spawn), ambience (object, ignored)

**Constraining decisions:**
- No behavior changes for valid area.json files — validation only rejects/warns on invalid data
- Vanilla JS only, no external validation libraries
- Tab-4 indentation, double quotes, semicolons

<task type="auto">
  <name>Task 1: Create validateAreaDefinition() with comprehensive schema validation</name>
  <files>js/AreaManager.js</files>
  <action>
  Add a `validateAreaDefinition(areaDef, areaId)` method to AreaManager. This method validates the parsed JSON object and throws on critical issues or warns on optional field problems.

  **Required field validation (throw Error on failure):**
  - `id`: must be a non-empty string
  - `name`: must be a non-empty string
  - `background`: must be a non-empty string
  - `width`: must be a positive number
  - `height`: must be a positive number

  Error format: `throw new Error(\`[AreaManager] Area '${areaId}': missing or invalid '${field}' (expected ${expected}, got ${typeof actual})\`);`

  **Optional field validation (console.warn on issues, do not throw):**

  - `blocked`: if present, must be an array. Each entry must have numeric `q` and `r`. Warn and filter out entries missing q/r.
  - `spawns`: if present, must be an object. Each value must have numeric `q` and `r`. Warn about malformed spawn points.
  - `npcs`: if present, must be an array. Each entry must have string `templateId` and numeric `hexQ`, `hexR`. Warn about entries missing required NPC fields.
  - `exits`: if present, must be an array. Each entry must have string `id`, array `hexes`, string `target`, string `spawn`. Warn about malformed exits.

  Warning format: `console.warn(\`[AreaManager] Area '${areaId}': ${field}[${index}] ${problem}\`);`

  Do NOT validate `ambience` (not used by game systems yet).

  Keep the method focused: validate structure, not game logic (e.g., don't check if blocked hexes are within world bounds — that's a future concern).
  </action>
  <verify>Read AreaManager.js and confirm validateAreaDefinition method exists with required and optional field checks.</verify>
  <done>validateAreaDefinition() method validates all required fields (throws on failure) and optional fields (warns on issues) with descriptive messages including area ID, field name, and expected type.</done>
