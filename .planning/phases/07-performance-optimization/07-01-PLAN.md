<objective>
Cache pathfinding results with version-based invalidation to eliminate redundant A* computations.

Purpose: Pathfinding is recomputed for identical start/goal/obstacle combinations — player re-clicking the same destination in exploration, or AI evaluating the same paths. A cache keyed on (start, goal) with a version counter for invalidation eliminates this waste.
Output: Pathfinding.js with result cache, invalidation wired into MovementSystem and AreaManager.
</objective>

<task type="auto">
  <name>Task 1: Add path result cache with version-based invalidation to Pathfinding.js</name>
  <files>js/Pathfinding.js</files>
  <action>
Add a caching layer to Pathfinding.findPath():

1. Add instance properties in constructor:
   - `this._pathCache = new Map()` — stores cached results
   - `this._cacheVersion = 0` — invalidation counter

2. Add `invalidateCache()` method that increments `this._cacheVersion` and clears `this._pathCache`.

3. In `findPath()`, before running A*:
   - Build cache key: `${this._cacheVersion}:${hexKey(start.q, start.r)}-${hexKey(goal.q, goal.r)}`
   - Check `this._pathCache.get(cacheKey)` — if hit, return the cached path (slice to return fresh array so caller mutations don't corrupt cache)
   - After A* computes a result, store it: `this._pathCache.set(cacheKey, result)`

4. Also cache empty results (failed paths) — prevents re-running expensive A* for unreachable destinations.

5. In `setBlockedHexes()`, call `this.invalidateCache()` — blocked hex changes mean all cached paths are potentially invalid.

Do NOT add an LRU eviction or size limit — the cache is cleared on every version bump (movement, area load), so it stays small. Adding complexity for bounded size is unnecessary overhead for this use case.
  </action>

Enter combat, run a turn — AI pathfinding should still work correctly. Toggle combat off, move again — paths should still be correct.
  </verify>
  <done>findPath() returns cached results for repeated calls with same start/goal between invalidations. setBlockedHexes() clears cache. All existing movement behavior unchanged.</done>
