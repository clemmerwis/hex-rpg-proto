Memoize AI distance calculations by building a per-turn distance matrix, eliminating redundant hexDistance iteration across multiple AI methods.

Purpose: AISystem calls hexDistance in findClosestEnemy, findNearestSameFaction, findNearestCharacter, and getMoveTowardAction — iterating the same character pairs multiple times per turn. A precomputed distance matrix built once per turn replaces all these per-method iterations with O(1) lookups.
Output: AISystem with per-turn distance matrix, wired from GameStateManager.processAITurns().
</objective>

<task type="auto">
  <name>Task 1: Add per-turn distance matrix to AISystem</name>
  <files>js/AISystem.js</files>
  <action>
Add a per-turn distance precomputation system:

1. Add instance property in constructor: `this._distanceMatrix = new Map()`.

2. Add `beginTurn(allCharacters)` method:
   - Clear `this._distanceMatrix`
   - Get living characters: `allCharacters.filter(c => !c.isDefeated)`
   - For each pair (i, j) where i < j in the living array:
     - Compute `this.hexGrid.hexDistance({q: chars[i].hexQ, r: chars[i].hexR}, {q: chars[j].hexQ, r: chars[j].hexR})`
     - Store both directions: key `${hexKey(chars[i].hexQ, chars[i].hexR)}-${hexKey(chars[j].hexQ, chars[j].hexR)}` → distance, and reverse key → same distance
   - This builds the full symmetric matrix in N*(N-1)/2 computations instead of repeated N iterations per method call

3. Add `_getDistance(charA, charB)` private method:
   - Build key: `${hexKey(charA.hexQ, charA.hexR)}-${hexKey(charB.hexQ, charB.hexR)}`
   - Return `this._distanceMatrix.get(key)` if found
   - Fallback: compute `this.hexGrid.hexDistance(...)` directly (defensive — handles characters not in the matrix, e.g., if beginTurn wasn't called)

4. Refactor these methods to use `_getDistance` instead of inline `this.hexGrid.hexDistance`:
   - `findNearestSameFaction()` — replace the hexDistance call with `this._getDistance(character, other)`
   - `findClosestEnemy()` — replace with `this._getDistance(character, enemy)`
   - `findNearestCharacter()` — replace with `this._getDistance(character, other)`
   - `getMoveTowardAction()` — replace the `distToTarget` hexDistance call with `this._getDistance(character, target)`
   - `getAIAction()` neutral clustering distance check — replace with `this._getDistance(character, clusterTarget)`

Do NOT touch hexDistance calls inside Pathfinding.js — those are part of A* heuristic and don't benefit from the matrix (different coordinate pairs each iteration).
  </action>

  </verify>
  <done>AISystem has beginTurn() that builds distance matrix. All 5 distance-using methods use _getDistance() for O(1) lookups. Fallback to direct computation if matrix miss. AI decisions identical to before.</done>
