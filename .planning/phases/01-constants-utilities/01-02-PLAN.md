<objective>
Create a shared hexKey(q, r) utility function in const.js and replace all ad-hoc `${q},${r}` hex key patterns across the codebase.

Purpose: Eliminate the fragile string convention where hex coordinates are converted to string keys via `${q},${r}` in 5+ files. A single canonical function prevents silent bugs from inconsistent key formats.
Output: const.js exports hexKey(). All Set/Map hex key lookups use hexKey() instead of inline template literals.
</objective>

Add a simple exported utility function near the top of const.js (after GAME_CONSTANTS, before the data tables):

```javascript
/**
 * Create a canonical string key from hex coordinates for Set/Map lookups.
 * Replaces ad-hoc `${q},${r}` patterns throughout the codebase.
 */
export function hexKey(q, r) {
    return `${q},${r}`;
}
```

This is intentionally minimal — just a function wrapping the string template. The value is single-source-of-truth, not abstraction.
