  <name>Task 1: Create HexGridRenderer.js with all hex drawing methods</name>
  <files>js/HexGridRenderer.js</files>
  <action>
Create js/HexGridRenderer.js as a new ES6 module class. Extract hex-grid-related rendering methods from Renderer.js:

**Constructor signature:**
```js
constructor(hexGrid, hexSize)
```

Store hexGrid and hexSize. Other dependencies injected via setDependencies() to match Renderer's pattern (these are set after construction in Game.js):

**setDependencies(deps):**
Store: game, getCharacterAtHex, gameStateManager, inputHandler, pathfinding, engagementManager

**Methods to extract from Renderer (move verbatim, then adapt `this.` references):**
- getFactionData(character) — faction color lookup (needed by drawFactionBorders and drawActiveHexGlow)
- getConnectedBlockedHexes(startQ, startR) — flood-fill for blocked hex highlighting
- drawHexGrid(ctx, cameraX, cameraY) — visible hex calculation + grid line pass + drawHex pass
- _drawHexPath(ctx, hexPoints, fillStyle, strokeStyle, lineWidth) — hex path drawing helper
- drawHex(ctx, q, r) — individual hex content (blocked overlay, active glow, faction borders, hover/selected/marked)
- drawFactionBorders(ctx, hexPoints, q, r, character) — faction borders with engagement gradients
- drawHoverHex(ctx, hexPoints) — blue hover highlight
- drawSelectedHex(ctx, hexPoints) — selected hex highlight
- drawMarkedHex(ctx, hexPoints) — orange marker highlight
- drawActiveHexGlow(ctx, hexPoints, center, character) — faction-colored glow effect

**Key change:** All draw methods receive `ctx` (canvas context) as first parameter instead of using `this.ctx`, since HexGridRenderer doesn't own the canvas. This keeps the canvas owned by Renderer.

Also move the flood-fill cache fields:
- _cachedConnectedBlockedHexes
- _cachedHoveredBlockedKey

Import: GAME_CONSTANTS, FACTIONS, hexKey from const.js; GAME_STATES, COMBAT_ACTIONS from GameStateManager.js

<task type="auto">
  <name>Task 2: Wire HexGridRenderer into Renderer and Game.js</name>
  <files>js/Game.js, js/Renderer.js</files>
  <action>
**In Game.js:**
1. Import HexGridRenderer
2. Create instance after hexGrid:
   ```js
   this.hexGridRenderer = new HexGridRenderer(this.hexGrid, GAME_CONSTANTS.HEX_SIZE);
   ```
3. In setupCallbacks(), after setting Renderer dependencies, set HexGridRenderer dependencies:
   ```js
   this.hexGridRenderer.setDependencies({
       game: this.state,
       getCharacterAtHex: this.getCharacterAtHex.bind(this),
       gameStateManager: this.gameStateManager,
       inputHandler: this.inputHandler,
       pathfinding: this.pathfinding,
       engagementManager: this.engagementManager
   });
   ```
4. Pass hexGridRenderer to Renderer via setDependencies:
   ```js
   this.renderer.setDependencies({ ..., hexGridRenderer: this.hexGridRenderer });
   ```

**In Renderer.js:**
1. Remove all extracted methods (getFactionData, getConnectedBlockedHexes, drawHexGrid, _drawHexPath, drawHex, drawFactionBorders, drawHoverHex, drawSelectedHex, drawMarkedHex, drawActiveHexGlow)
2. Remove flood-fill cache fields from constructor
3. Remove imports no longer needed (hexKey, GAME_STATES, COMBAT_ACTIONS if only used by extracted methods)
4. Store hexGridRenderer from setDependencies
5. In render(), replace `this.drawHexGrid(cameraX, cameraY)` with:
   ```js
   this.hexGridRenderer.drawHexGrid(this.ctx, cameraX, cameraY);
   ```

**Note:** getFactionData is also used by drawCharacter and drawNameplate (character rendering). For now, add a simple getFactionData method to Renderer that delegates to hexGridRenderer:
```js
getFactionData(character) {
    return this.hexGridRenderer.getFactionData(character);
}
```
This will be cleaned up in 04-02 when character rendering is extracted.
