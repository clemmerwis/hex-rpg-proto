**Status:** SHIPPED 2026-02-17
**Phases:** 1-7

## Overview

A systematic code quality pass transforming the hex-grid RPG prototype from a working-but-messy state into a well-structured, maintainable codebase. Starting with foundational cleanup (constants, utilities), progressing through major file decompositions (CombatSystem, GameStateManager, Renderer), then smaller file splits, bug fixes, and finally performance optimizations.

## Milestone Summary

**Key Decisions:**
- Focus on code quality over features (codebase growing harder to maintain)
- No test infrastructure in scope (separate milestone)
- Preserve all existing behavior (refactoring must be safe)
- hexKey() as single source of truth for hex coordinate keys
- Direct DI injection for cross-cutting concerns (EngagementManager)
- Cache invalidation by world dimensions, not camera position
- Per-turn distance matrix with no explicit cleanup (cleared at next beginTurn)

**Issues Resolved:**
- MovementSystem setTimeout race condition replaced with deferred callback queue
- area.json loaded without validation — added schema validation with descriptive errors
- DI wiring unvalidated — added constructor and post-wiring validation to all modules
- Hex visibility recalculated every frame — cached with dimension-based invalidation
- Flood-fill computed per hover — pre-computed on area load with O(1) lookup
- Pathfinding recomputed for identical paths — cached with version-based invalidation
- AI distance calculations repeated per method — memoized via per-turn distance matrix

**Issues Deferred:**
None.

**Technical Debt Incurred:**
None — this milestone was specifically about reducing technical debt.
